📢 GSS v6.6 更新聲明：關於「成人內容」標籤的真相

致各位 GSS 用戶：

這是一個極其荒謬的版本。從今天起，您會在 Chrome 商店看到 GSS 被標記為「成人內容」。身為獨立開發者，我認為有義務向各位誠實交代這個決策背後的審核機制。


🔍 消失的邏輯：從「核准」到「違規」

這並非 GSS 包含了任何不雅內容，而是因為 Google 審核標準的反覆無常：

v6.1 版本：早已通過審核，當時完整支援 W.TV 且被判定為全年齡。

v6.2 版本：常規更新，審核依然順利通過。

v6.3 至今：Google 在無預警、無溝通的情況下，突然將 W.TV 定義為成人網域，並以此卡住 GSS 的所有技術更新。

👤 團隊現況：產能應用於創造，而非虛耗於審核

GSS 的開發團隊只有我一個人。我的產能應該花在優化貼圖體驗，而不是浪費在應對繁瑣的審核流程和漫長的等待時間。與其花時間去規避審核系統，我寧願選擇「吃下這隻死貓」，換取開發功能的自由。


🚀 更新策略調整與階段性停更說明

由於 Google Chrome Web Store 的審核機制極度緩慢且充滿不確定性，為了確保版本資訊的統一，往後我將採取同步上載至 Chrome、Firefox、Edge 與 GitHub 的策略。

⚠️ 版本提醒：如果您看到的版本號不是 v6.6，代表該平台的審核尚未通過。

在此我必須公開感謝 Firefox (Mozilla) 團隊對開發者的信任與高效審查——在那裡，開發者的產能是被尊重的。

同時，這或許是近期最後一次同步更新，我打算暫停更新一段時間，主要基於以下原因：

  • 一、注意力分配問題：自從開發 GSS 以來，我幾乎所有時間腦袋裡都在想著如何改善它，我現在沒法好好分配自己的注意力，我需要停下來休息。

目前的版本我已經盡力做好了。雖然不知道還有沒有 bug，但我暫時不想寫了。如果以後環境變好，或者我找到了新的好用工具，或許會再回來。

v6.1 버전: 오래전 심사를 통과했으며, 당시 W.TV를 완전 지원하고 전연령으로 판정되었습니다.

v6.2 버전: 정규 업데이트, 심사는 여전히 순조롭게 통과했습니다.

v6.3부터 현재: Google이 예고도 없고, 소통도 없이 갑자기 W.TV를 성인 도메인으로 정의하고, 이를 이용해 GSS의 모든 기술 업데이트를 막고 있습니다.

그들의 논리에서는: "해당 도메인의 기술 적응(채팅룸 좌표 위치 지정) = 성인 콘텐츠 제공". 이 "도구"와 "환경"을 혼동하는 터무니없는 판단은 제가 받아들일 수 없습니다.

GSS 개발팀은 저 혼자입니다. 저의 생산성은 스티커 경험을 최적화하는 데 써야 하며, 번거로운 심사 절차와 긴 대기 시간에 낭비해서는 안 됩니다. 심사 시스템을 회피하는 시간을 쓰는 것보다, 기능을 개발하는 자유와 교환하여 "불명예스러운 결정을 내릴 수밖에 없었습니다" 것을 선택하겠습니다.

  • 3. 번거로운 심사 메커니즘: Google의 심사는 지나치게 간섭이 심하며, 이미 폐쇄된 플랫폼인 DLive와 관련된 기능까지 제거하라고 요구합니다. 이러한 반복적인 심사 대응에 매우 지쳐 있는 상태입니다.

현재 버전은 제가 할 수 있는 최선을 다했습니다. 아직 버그가 남아있을지도 모르지만, 당분간은 더 이상 코드를 작성하고 싶지 않습니다. 향후 환경이 개선되거나 새롭고 유용한 도구를 찾게 된다면 다시 돌아올 수도 있습니다.

버전 6.6의 알려진 문제: 메인 화면의 ID 입력란에 데이터가 저장되지 않습니다. 갤러리 편집기를 사용하여 ID를 추가해 주세요. 죄송합니다.

🔓 기술 개방: GSS Sticker Protocol

GSS의 이미지 변환 로직을 "GSS Sticker Protocol"으로 공식적으로 명명합니다. 저는 어떤 개발자든 이 로직을 작품에서 참조하여 분산형 스티커 방식을 공동으로 추진하는 것을 환영합니다.

앞으로 모두가 동일한 프로토콜을 사용하면, 어떤 플랫폼, 어떤 도구를 사용하든, 모두 서로의 스티커를 볼 수 있습니다. 기술 프로토콜의 개방을 통해, 스티커 사용이 단일 플랫폼의 심사 메커니즘에 제한되지 않도록 합니다.
